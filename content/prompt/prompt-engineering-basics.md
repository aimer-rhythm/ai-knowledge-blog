# 提示工程入门

提示工程（Prompt Engineering）是与大语言模型交互的核心技能。好的提示可以显著提升模型的输出质量，而差的提示可能导致不相关或错误的回答。

## 什么是提示工程？

提示工程是设计和优化输入提示（prompt）的过程，目的是引导大语言模型产生期望的输出。这不仅仅是"问问题"——它是一门结合了语言学、认知科学和软件工程的实践技能。

## 基础提示技巧

### 1. 明确具体

```text
❌ 差的提示：
"告诉我关于 Python 的东西"

✅ 好的提示：
"请列出 Python 3.12 的 5 个最重要的新特性，
每个特性包含：名称、简要描述、代码示例。"
```

### 2. 提供上下文

```text
✅ 带上下文的提示：
"我是一个有 3 年 Java 经验的后端开发者，正在学习 Python。
请用 Java 开发者能理解的方式解释 Python 的装饰器模式，
并与 Java 的注解进行对比。"
```

### 3. 指定输出格式

```text
✅ 指定格式：
"请以 JSON 格式输出以下数据：
- 字段：name, category, difficulty (1-5), description
- 主题：常见的机器学习算法
- 数量：5个"
```

## 高级提示技术

### Chain-of-Thought（思维链）

思维链提示让模型展示推理过程，适合需要逻辑推理的任务：

```text
问题：一个商店有 23 个苹果。上午卖了 8 个，下午又进了 15 个，
然后又卖了 12 个。现在有多少个苹果？

请一步一步思考：
1. 开始：23 个苹果
2. 上午卖了 8 个：23 - 8 = 15
3. 下午进了 15 个：15 + 15 = 30
4. 又卖了 12 个：30 - 12 = 18

答案：18 个苹果
```

::callout{type="tip"}
在提示中加入"让我们一步一步思考"（Let's think step by step）这样的简单指令，就能显著提升模型在数学和逻辑推理任务上的表现。
::

### Few-Shot Learning（少样本学习）

通过提供几个示例来教模型理解你期望的输出模式：

```text
将以下句子分类为"积极"或"消极"：

句子：这部电影太精彩了！
分类：积极

句子：服务态度很差。
分类：消极

句子：今天的天气让人心情愉快。
分类：
```

### 角色设定

```text
你是一位资深的数据科学家，拥有 10 年的机器学习经验。
你的沟通风格是：
- 技术准确但通俗易懂
- 喜欢用实际案例说明概念
- 会指出常见的误区和陷阱

请解释什么是过拟合，以及如何避免它。
```

## 提示工程框架

### CRISPE 框架

| 元素 | 说明 | 示例 |
|------|------|------|
| **C**apacity | 角色能力 | "你是 Python 专家" |
| **R**ole | 具体角色 | "作为代码审查者" |
| **I**nsight | 背景信息 | "这是一个 Flask 项目" |
| **S**tatement | 具体任务 | "审查这段代码的安全性" |
| **P**ersonality | 表达风格 | "用简洁的要点列出" |
| **E**xperiment | 输出要求 | "给出 3 个改进建议" |

## 常见陷阱

::callout{type="warning"}
**避免以下常见错误：**
- 提示过于模糊或宽泛
- 一次性要求太多不同的任务
- 没有指定输出格式或长度限制
- 忽略模型的知识截止日期
- 假设模型能"记住"之前的对话（在 API 调用中）
::

## 实战示例：代码审查提示

```text
请审查以下 Python 代码，关注这些方面：
1. 安全性漏洞
2. 性能问题
3. 代码可读性
4. 最佳实践

对每个发现的问题：
- 标注严重程度（高/中/低）
- 解释为什么这是问题
- 提供修复建议和代码示例

代码如下：
[粘贴代码]
```

## 总结

提示工程的核心原则：

1. **清晰明确**：明确说明你想要什么
2. **提供上下文**：给模型足够的背景信息
3. **结构化**：使用格式化的提示结构
4. **迭代优化**：根据输出调整提示
5. **设定约束**：限制输出长度、格式、范围

掌握这些技巧将让你在使用 AI 工具时事半功倍。
